# 项目报告计划与分工

## 一、项目整体进展总结

### 1.1 项目概述
本项目是一个**基于FunSearch进化算法和LSTM+PPO强化学习的投资组合优化系统**，旨在通过自动化策略搜索和强化学习协同推理，实现比传统量化策略更优的投资组合配置。

### 1.2 已完成功能

#### 核心系统架构
- **FunSearch进化引擎**：多Island并行策略进化，已成功集成GPT-4o-mini API
- **LSTM+PPO动态RL系统**：基于强化学习的动态权重调整，每日重新平衡
- **回测框架**：支持多种策略对比（等权、最小方差、最大夏普、LSTM+PPO动态RL、FunSearch）

#### 技术实现
- ✅ 全链路数据流：多因子数据 → 策略生成 → 回测评估 → 结果可视化
- ✅ 策略进化机制：10个Island并行进化，每轮评估后保留最佳策略
- ✅ 进化记录系统：完整的JSON日志记录每代策略代码和性能指标
- ✅ 报告生成系统：自动生成Markdown和HTML格式的绩效报告
- ✅ 多维度可视化：净值曲线、回撤曲线、雷达图、权重分布图

#### 性能表现（当前状态）
| 策略 | 夏普比率 | 索提诺比率 | 最大回撤 | 最终净值 |
|------|---------|-----------|---------|---------|
| 等权 | 0.78 | 1.34 | -0.25 | 14.7M |
| 最小方差 | 0.85 | 1.45 | -0.22 | 15.4M |
| 最大夏普 | 0.88 | 1.50 | -0.23 | 15.7M |
| LSTM+PPO动态RL | ~2.5+ | ~4.0+ | ~-0.15 | ~60M+ |
| FunSearch | ~0.85 | ~1.45 | ~-0.23 | ~15.5M |

**结论**：FunSearch策略表现接近最小方差/最大夏普，但未超越动态RL系统。

### 1.3 存在的问题

**FunSearch表现一般的核心原因**：
1. LLM生成策略是从零开始的设计，缺乏经典量化理论支撑
2. 进化过程中未充分利用历史协方差矩阵和收益率估计
3. 策略空间局限于简单加权平均和随机扰动

**下一步优化方向**：
- 在经典量化策略框架（最小方差、最大夏普）基础上进化
- 引入参数优化和条件策略切换机制
